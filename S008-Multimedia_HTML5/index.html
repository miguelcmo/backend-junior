Imágenes, audio, vídeo y SVG en HTML5

Ejemplos prácticos usando archivos de multimedia-portfolio/assets/.

Imágenes

Comparación de formatos y tamaños. Observe el atributo alt, y el uso de width/height para ayudar al navegador a reservar espacio.

Imagen responsiva (ejemplo con picture y source)

El elemento picture permite elegir formato/versión según soporte o medios.

Consejos rápidos

  • Optimizar para la resolución de destino: imágenes grandes afectan rendimiento.
  • Preferir WebP/AVIF cuando sea posible para mejor compresión.
  • Siempre incluir alt descriptivo para accesibilidad.

Audio

Use <audio controls> y provea múltiples <source> con distintos formatos para compatibilidad.

Implicaciones: formatos comprimidos (MP3/OGG) son preferibles para la web; WAV ocupa mucho y tarda en cargar.

Vídeo

El elemento <video controls> admite múltiples <source>. Mostrar varios tamaños/resoluciones ayuda a entender consumo de datos.

Recuerda: ofrecer versiones adaptadas (resoluciones/bitrates) y usar streaming o CDN para producción.

SVG

SVG es vectorial: escala sin pérdida. Ideal para iconos y gráficos simples.

SVG inline (ejemplo minimal)

SVG es pequeño para íconos y permite manipulación con CSS/JS si se inserta inline.

Implicaciones de diseño

  • Rendimiento: archivos grandes (vídeo, PNG, WAV) aumentan tiempo de carga; optimice y ofrezca versiones.
  • Responsividad: use srcset/picture para servir imágenes adecuadas al dispositivo.
  • Accesibilidad: siempre provea alt, subtítulos/transcripciones para audio/vídeo.
  • Formatos: WebP/AVIF para imágenes, MP3/OGG para audio y MP4/WebM para vídeo (según compatibilidad).
